svgobjects 2.0

What's New

Introduction

An overview of svgbjects 2.0 can be found at http://www.svgobjects2.com. This document outlines the new additions since the 1.0 distribution.

Most notable is the move to Eclipse projects for WebObjects 5.4. The following are new to svgobjects 2.0:

DirectToiPhone

A D2W framework for generating iPhone web apps (in SVG and XHTML+CSS).

DirectToWeb 'iPod Look' framework

This is a D2W framework for presenting an iPod like UI to a DirectToWeb application. The interface 100% SVG: a rich interface application or a web 2.0 app.

It is best viewed in WebKit (aka Safari 3.0), or FireFox 1.5+, Opera 8+ or other SVG native browsers. That however excludes any existing version of IE (without a SVG plugin).

Example/Demo: DirectToiPhone

This example is a music store web app: in an SVG or XHTML+CSS 'look'.

CoreStore Framework for model and business logic for demo music store.
StoreAdmin Simple D2W admin app for CoreStore.
SVGMusicStore A SVG front end to CoreStore that mimics an iPod interface. It is a true D2W application that employs DirectToiPhone and the iPod Look framework to generate its interface.
XHTMLMusicStore An iPhone front end to CoreStore that is "API-compatible" with the SVG / 'iPod Look' SVG music store.

sXBL Examples

These examples need to be viewed with FireFox 1.5 or better. sXBL is currently unsupported in Safari or WebKit.

SOTriangleTest Test application binding a <triangle> to SVG with interactivity.
SXShapeTest Simple test app binding <shape> to SVG again with interactivity.

1.0 Deprecations

All trivial SVGObjects 1.0 dynamic elements have been deprecated. This is because the WebObjects 5.4 combined/XML template parser can shortcut such SVG dynamic elements with a WOGenericContainer/Element.
